# Fitness

A personal single-user workout PWA — schedule, log, and track resistance and athletic training. Built with Next.js 14 (App Router), TypeScript (strict), Tailwind, and a local SQLite store via `better-sqlite3`. Installable/offline-capable via `next-pwa`.

See [`SPEC.md`](./SPEC.md) for the product/behavior specification and [`MEDIA_SOURCING.md`](./MEDIA_SOURCING.md) for exercise-media provenance.

## Stack

- **Next.js 14** (App Router, React Server Components + server actions)
- **TypeScript** (`strict: true`)
- **better-sqlite3** — synchronous embedded SQLite; schema + ad-hoc column migrations in `lib/db.ts`
- **Tailwind CSS**
- **next-pwa** — service worker / offline shell (build output, not committed)
- **node:test** — test runner (`.test.cjs` files with an on-the-fly TypeScript transpile loader)

## Requirements

- **Node 22 LTS.** `better-sqlite3` compiles a native addon that does **not** build against Node 26+. Use a version manager:
  ```bash
  fnm use 22    # or: nvm use 22
  ```

## Dev setup

```bash
npm ci                 # install exact dependencies
npm run dev            # dev server on http://localhost:3102 (shared data dir)
npm run dev:sandbox    # dev server on :3112 with an isolated ./dev-data store
```

`FITNESS_DATA_DIR` controls where the SQLite database lives. `dev:sandbox` points it at `./dev-data` so you can experiment without touching real data. Unset, it defaults to the app's standard data location.

## Common commands

```bash
npm run build          # production build (also regenerates the PWA service worker)
npm start              # serve the production build on :3102
npm test               # run the node:test suite
npm run lint           # ESLint (next/core-web-vitals)
npm run seed           # seed the exercise catalog / sample data (scripts/seed.mjs)
npx tsc --noEmit       # type-check only
```

## Deployment

The app serves on **port 3102**. `scripts/setup-tunnel.sh` sets up external access. The service worker and workbox assets under `public/` are **build artifacts** (regenerated by `npm run build`) and are not tracked in git.

## Testing notes

Tests use `node:test` and load TypeScript modules through a shared transpile loader. Run the whole suite with `npm test`. Plan-generation tests are being refactored toward pure/injected inputs so they can run without a compiled `better-sqlite3`.
